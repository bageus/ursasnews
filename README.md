# ursasnews

Минимальный каркас внутреннего инструмента Ursas News с фокусом на **Фазу 2 (генерация ролика)**.

## Что сейчас есть

- Простое табовое UI (`index.html`):
  - Лента (ручной отбор новостей для отладки),
  - Генерация ролика,
  - Рубрики,
  - Настройки.
- Базовая логика MVP (`app.js`):
  - добавление новостей вручную,
  - очередь новостей в выпуск,
  - сборка JSON-сценария для preview/final render.
- Базовые стили (`styles.css`) в минималистичном internal-дизайне.

## Запуск

Откройте `index.html` в браузере.

Для локального HTTP-сервера можно использовать:

```bash
python3 -m http.server 8080
```

Затем открыть `http://localhost:8080`.

## Деплой на Vercel

В проект добавлен минимальный `vercel.json` (`version: 2` + `rewrites`) с fallback на `index.html`, чтобы корневой URL и клиентские пути не отдавали 404 при SPA-навигации.

## Документация

- `PIPLAN.md` — подробный markdown-план реализации с задачами, путями решения и roadmap.
- В форме генерации ролика можно задать параметры субтитров: `language`, `font_family`, `font_weight` (попадают в `render.subtitles_style` payload).
- В основной форме выпуска есть блок «Новости для чтения»: поля `Новость 1..N` с кнопкой добавления, у каждого поля есть короткий заголовок, текст, ссылка, загрузка изображения и счетчик `символы / слова`.
- Заголовок и изображение новости выводятся на `board-news` в превью сцены.

## Формат рта (lip-sync) в текущем MVP

Сейчас используется **спрайт-листовый формат** 100x100:

- `public/mounth/рот закрыт нейтральный.webp` — 10 кадров закрытого/нейтрального рта;
- `public/mounth/рот улыбка с языком.webp` — 8 кадров открытого рта (слева направо по убыванию размера).
- `public/mounth/улыбка зубами.webp` — idle-состояния рта (используются в начале и конце речи).

Логика анимации:

- кадры выбираются случайно с учетом текста;
- короткие слова — маленький рот, обычные — средний, при `!` — большой;
- паузы учитываются на `,` и `.`;
- темп задается либо общей длительностью чтения, либо скоростью слов/мин.
- закрытый рот рендерится в масштабе 70% (`scale: 0.7`) и выравнивается по общему центру/уровню с открытым ртом через смещения `offset_x/offset_y`.
- перед стартом речи и после ее завершения устанавливается случайный кадр из `улыбка зубами.webp` как базовое состояние рта.
