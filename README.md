# Ursas News — Internal MVP

Веб-инструмент для редактора/продюсера **Ursas News**, который помогает вручную собрать выпуск, подготовить текст под озвучку, превьюнуть сцену с lip-sync, настроить рубрики и получить JSON payload для дальнейшего рендера видео.

Проект сейчас находится в формате **front-end MVP без сборки**: статические HTML/CSS/JS-файлы, запуск прямо в браузере или через простой HTTP-сервер.

---

## 1) Назначение проекта

`ursasnews` закрывает рабочий сценарий «от новости до сценария выпуска»:

1. Редактор добавляет новости вручную в очередь.
2. Вкладка генерации собирает структуру выпуска: интро, блоки новостей, аутро, субтитры и параметры сцены.
3. Оператор запускает preview, чтобы проверить поведение рта (lip-sync), субтитров и слоёв сцены.
4. На выходе формируется JSON-объект сценария (для preview или final render), который можно отправлять в дальнейший pipeline генерации ролика.

## 3) Основные возможности

### 3.1 Лента (ручной отбор)

- Добавление новости вручную:
  - заголовок,
  - ссылка,
  - описание,
  - источник,
  - тег,
  - тональность (bullish/bearish/neutral),
  - сила сигнала,
  - картинка.
- Очередь новостей для выпуска.
- Блок мониторинга источников:
  - платформы (X, Reddit, Telegram, Instagram, Reuters, Bloomberg, CoinDesk, Grok, Gemini),
  - хранение целей мониторинга,
  - парсинг «негатива за 24ч» для Reuters/Bloomberg/CoinDesk по RSS.

### 3.2 Генерация ролика

- Сборка структуры выпуска:
  - название,
  - формат кадра (vertical/horizontal/square),
  - fps,
  - режим чтения (фиксированная длительность или слова/мин),
  - субтитры (язык/шрифт/жирность/размер/позиция через джойстик),
  - интро и аутро,
  - динамический список новостей «для чтения». 
- Для каждой новости:
  - заголовок, текст, ссылка, изображение,
  - настройка кадрирования/позиционирования изображения,
  - настройки позиции и размера заголовка на доске,
  - выбор и подтверждение сцен (таймлайн превью).
- Preview режима:
  - анимация рта,
  - подсветка/отрисовка субтитров,
  - переключение доски между базовым и news-режимом,
  - выполнение текстовых команд (`*speak_pause`, `*board_news_front`, `*board_news_back` и др.).

### 3.3 Рубрики

- Каталог рубрик (Ursas Index, Daily Pick, Top Gainers/Losers, Stocks Movers, Crypto Bubbles и др.).
- Добавление рубрик в выпуск.
- Описание рубрик на RU/EN с сохранением в `localStorage`.
- Просмотр карточек рубрик в оверлее.
- Фильтры для crypto gainers/losers (валюта, капитализация, ликвидность, исключение стейблов).

### 3.4 Настройки/данные рынка

- Топ gainers/losers по крипте через CoinGecko.
- Movers по акциям через FinancialModelingPrep (нужен рабочий API-ключ).
- Ссылки/iframe-режим для Crypto Bubbles (с учётом ограничений `X-Frame-Options`).
- Daily Pick (number-of-day) с flip-анимацией и пересчётом по CET.

## 4) Архитектура проекта

Проект без bundler'а и без framework:

- `index.html` — разметка интерфейса.
- `styles.css` — стили и анимации.
- `app.js` — основной orchestration-скрипт (UI, состояния, preview, генерация payload).
- `js/tab-navigation.js` — переключение вкладок.
- `js/market-movers.js` — контроллер рыночных рубрик и фильтров.
- `js/number-of-day.js` — контроллер flip-табло Daily Pick.
- `js/app-config.js` — конфигурация приложения (провайдеры, рубрики, команды, ассеты рта и т.п.).
- `public/` — графические ассеты сцены (фон, доска, рот, руки, голова и др.).

Подключение модулей идёт напрямую через `<script>` в HTML, экспорт через глобальный `window.*` namespace.

## 5) Внешние API и интеграции

Используются клиентские запросы из браузера:

- `https://api.ursasstube.fun/api/leaderboard/top` — leaderboard для рубрики Ursass Tube.
- `https://api.coingecko.com/api/v3` — данные по крипто-рынку (gainers/losers).
- `https://financialmodelingprep.com/api/v3` — market movers по акциям (через API key).
- RSS Reuters/Bloomberg/CoinDesk — для автодобавления негативных новостей за 24 часа.

> Важно: часть внешних источников может быть ограничена CORS/анти-бот защитой; в MVP предусмотрены fallback-сценарии и статус-сообщения в UI.

---

## 6) Формируемые данные (payload)

Кнопки **Preview** и **Final Render** в форме выпуска формируют структуру сценария (JSON), в которую входят:

- метаданные выпуска,
- рендер-параметры (формат, fps, субтитры),
- итоговый speech-текст (интро + новости + аутро),
- массив новостей с `scene_settings` и `scene_frames`,
- выбранные рубрики,
- управляющие команды/паузы для тайминга превью.

Этот JSON является контрактом между UI и следующим этапом видеопайплайна.

## 7) Хранение состояния

Используется `localStorage` для пользовательских настроек в браузере:

- описания рубрик (`RU/EN`),
- фильтры market movers,
- часть пользовательских параметров интерфейса.

Это локальное хранение, без синхронизации между устройствами.

В репозитории есть `vercel.json` с rewrite на `index.html` для SPA/fallback-навигации.

Базовый процесс:

1. Импортировать репозиторий в Vercel.
2. Убедиться, что включён static deploy.
3. Проверить, что маршруты отдаются через fallback на `index.html`.

---

## 10) Ограничения MVP

- Нет серверной авторизации/ролей.
- Нет бекенд-очереди рендера прямо в этом проекте.
- Нет unit/integration/e2e тестов в текущей версии.
- Есть зависимость от доступности внешних API и их лимитов.
- Часть команд анимации помечена как «ассеты позже» и работает в упрощённом виде.

## 13) Для разработчиков

- Это vanilla JS кодовая база, поэтому дисциплина структуры и именования критична.
- При добавлении новых рубрик лучше сначала расширять `js/app-config.js`, затем `app.js`, и только потом UI-стили.
- Для новых внешних интеграций желательно сразу закладывать:
  - обработку таймаутов,
  - fallback-сообщения в интерфейсе,
  - кеширование/дебаунс запросов.
