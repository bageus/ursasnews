# ursasnews

Минимальный каркас внутреннего инструмента Ursas News с фокусом на **Фазу 2 (генерация ролика)**.

## Что сейчас есть

- Простое табовое UI (`index.html`):
  - Лента (ручной отбор новостей для отладки),
  - Генерация ролика,
  - Рубрики,
  - Настройки.
- Базовая логика MVP (`app.js`):
  - добавление новостей вручную,
  - очередь новостей в выпуск,
  - сборка JSON-сценария для preview/final render.
- Базовые стили (`styles.css`) в минималистичном internal-дизайне.

## Запуск

Откройте `index.html` в браузере.

Для локального HTTP-сервера можно использовать:

```bash
python3 -m http.server 8080
```

Затем открыть `http://localhost:8080`.

## Формат кадров рта (lip-sync)

Шаблон вида `public/base scene/mouth/mouth-{frame}.webp` означает **последовательность отдельных файлов**, а не атлас.

Рекомендуемый формат исходников:

- Размер каждого файла рта такой же, как у сцены (например 1080x1920).
- Фон прозрачный (alpha), виден только рот.
- Рот размещается в одних и тех же пиксельных координатах на каждом кадре (не сдвигать между файлами).
- Именование по порядку: `mouth-0001.webp`, `mouth-0002.webp`, ... `mouth-0100.webp`.

Почему так:

- в текущем MVP слой рта просто подменяет URL кадров по таймеру;
- если позиция рта фиксирована по пикселям, не нужно считать интервалы или координаты атласа;
- проще экспортировать и проверять художнику/дизайнеру.
